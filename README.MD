# Chatify

Real-time messaging, beautifully simple. A full-stack chat application with custom authentication, live presence indicators, image sharing, and a warm, distinctive UI.

## Features

- **Real-time messaging** вЂ” instant delivery via Socket.IO
- **Custom JWT auth** вЂ” httpOnly cookies, no third-party auth dependency
- **Online/offline presence** вЂ” live indicators across contacts and conversations
- **Image sharing** вЂ” upload and send images inline via Cloudinary
- **Profile photos** вЂ” upload and change your avatar anytime
- **Notification sounds** вЂ” keystroke and message sounds with a toggle
- **Welcome emails** вЂ” branded onboarding email sent via Resend (optional)
- **Rate limiting & bot protection** вЂ” powered by Arcjet (optional)
- **Optimistic messaging** вЂ” messages appear instantly before server confirmation
- **Responsive design** вЂ” sidebar collapses on mobile with back navigation
- **Keyboard shortcuts** вЂ” ESC to close a conversation

## Getting Started

### 1. Clone & install

```bash
git clone <your-repo-url>
cd chatify
npm run build    # installs both backend + frontend deps, builds frontend
```

### 2. Configure environment

```bash
cp backend/.env.example backend/.env
```

Edit `backend/.env` and fill in the required values:

| Variable | Required | Description |
|----------|----------|-------------|
| `MONGO_URI` | вњ… | MongoDB connection string |
| `JWT_SECRET` | вњ… | Random secret for signing tokens |
| `CLOUDINARY_CLOUD_NAME` | вњ… | Cloudinary dashboard в†’ cloud name |
| `CLOUDINARY_API_KEY` | вњ… | Cloudinary dashboard в†’ API key |
| `CLOUDINARY_API_SECRET` | вњ… | Cloudinary dashboard в†’ API secret |
| `CLIENT_URL` | вЂ” | Frontend URL (default: `http://localhost:5173`) |
| `PORT` | вЂ” | Backend port (default: `3000`) |
| `NODE_ENV` | вЂ” | `development` or `production` |
| `RESEND_API_KEY` | вЂ” | Optional вЂ” enables welcome emails |
| `EMAIL_FROM` | вЂ” | Sender address for emails |
| `EMAIL_FROM_NAME` | вЂ” | Sender display name (default: `Chatify`) |
| `ARCJET_KEY` | вЂ” | Optional вЂ” enables rate limiting |

**Generate a JWT secret:**

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### 3. MongoDB setup

**Local MongoDB:**
```bash
# macOS (Homebrew)
brew services start mongodb-community
# Use: MONGO_URI=mongodb://localhost:27017/chatify
```

**MongoDB Atlas (cloud):**
1. Create a free cluster at [mongodb.com/atlas](https://www.mongodb.com/atlas)
2. Add your IP to the whitelist (or use `0.0.0.0/0` for development)
3. Create a database user
4. Copy the connection string в†’ set as `MONGO_URI`

### 4. Run locally

```bash
# Terminal 1 вЂ” Backend (port 3000)
npm run dev:backend

# Terminal 2 вЂ” Frontend (port 5173)
npm run dev:frontend
```

### 5. Production build & start

```bash
npm run build   # installs deps + builds frontend
npm start       # starts backend, serves frontend from dist/
```

The app will be available at `http://localhost:3000`.

## Deployment

This project is ready for single-server deployment on platforms like [Render](https://render.com), [Railway](https://railway.app), or [Sevalla](https://sevalla.com):

1. Set the **build command** to `npm run build`
2. Set the **start command** to `npm start`
3. Configure all required env vars in the platform's dashboard
4. Set `NODE_ENV=production`
5. Set `CLIENT_URL` to your deployed URL (or leave unset вЂ” production serves the frontend from the same origin)

The backend will serve the built frontend assets in production mode automatically.

### Keep-alive health checks

The API now exposes `GET /api/health` for uptime checks.

If you use GitHub Actions, add a repository secret named `KEEP_ALIVE_URL` with your deployed health-check URL, for example:

```bash
https://your-backend.example.com/api/health
```

The bundled workflow at `.github/workflows/keep-alive.yml` pings that endpoint every 12 minutes so low-traffic deployments are less likely to idle.
